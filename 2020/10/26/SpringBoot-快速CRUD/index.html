国际化

在 Spring Boot 之前，要想实现国际化，操作非常繁琐。

  • 编写国际化配置文件，包含所有需要进行国际化的内容
  • 使用 ResourceBundleMessageSource 管理国际化资源文件
  • 在页面中使用 fmt:message 去除国际化内容

在 Spring Boot 中实现国际化。

  • 编写国际化配置文件

  • Spring Boot 自动配置好了管理国际化资源文件的组件

  • 在页面中引用国际化的值

以上配置只适用于浏览器切换显示语言时生效，我们希望能够通过按钮点击实现国际化，这就需要在按钮点击时，能够获取到 locale（区域信息对象）

Spring Boot 自动配置提供了 LocaleResolver （获取区域信息对象）

通过按钮点击发送请求当前页面并携带对应的区域信息。如何生效？编写新的区域信息解析器替换掉自动配置的。

删除 delxxx?id=xx xxx/{id} —> DELETE

公共片段抽取

Thymeleaf 提供了公共片段抽取，抽取出经常需要包含其他模板中的部分，例如页脚，页眉，菜单等部分。

根据 Thymeleaf 官方提供的引入公共片段的方式：

  • th:insert：将指定片段作为 host 标签的主体插入
  • th:replace：将 host 标签替换成指定的片段
  • th:include：不插入片段，而只插入片段内容

处理 PUT&DELETE 请求

Controller 无法直接处理 PUT&DELETE 的请求，需要在容器中添加相关组件

  • Spring MVC 容器中添加 HiddenHttpMethodFilter 组件
  • 前端发送 POST 请求
  • 创建一个 input 项，name 为 _method ，value 为要发送的 Http 动词

日志框架

​ 市面上的常见日志框架

JUL、JCL、Log4j、JBoss-logging、Log4j2、LogBack、Alf4j

​ 日志门面：SLF4j 日志实现：LogBack

Spring Boot 的底层是 Spring 框架，日志框架是 Commons-logging（JCL）

而Spring Boot 选用的就是 Slf4j + LogBack

Slf4j

使用

以后开发的时候，日志记录方法的调用，不应该来直接调用日志的实现类，而是调用日志抽象层里面的方法；

给系统里面导入slf4j的jar和 logback的实现jar。Slf4j官方使用手册

日志实现图示

每一个日志地实现框架都有自己的配置文件，使用 Slf4j 之后，配置文件仍需要做成对应的日志实现框架的配置文件。

Spring Boot 的日志关系

​ Spring Boot 利用 spring-boot-starter-logging 做日志功能

底层依赖关系

  • Spring Boot 底层也是使用的 Slf4j + LogBack 做日志记录
  • Spring Boot 也把其它的日志转成了 Slf4j，例如 log4j-to-slf4j
  • 如果要引入其它框架，一定要把这些框架默认的日志框架给排除掉

​ 给类路径下配置对应的客制化配置文件，Spring Boot 就不使用默认的日志配置了

logback.xml：直接被 spring boot 识别

logback-spring.xml：日志框架不会直接加载日志的配置项，由 SpringBoot 解析配置，使用高级 Profile 功能
